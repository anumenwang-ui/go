# 个人生活助理看板 V1 产品设计

日期：2026-05-15

## 1. 产品定位

这是一个极简、优雅、有温度的个人生活助理看板。它不是普通待办清单，也不是复杂项目管理工具，而是一个每天打开后能让用户快速获得生活秩序感的私人驾驶舱。

V1 的核心目标是回答四个问题：

1. 今天我最该做什么？
2. 今天有什么轻量的精神入口？
3. 我最近在滋养什么？
4. 我有哪些正在消耗预算的订阅？

产品方向采用“静谧生活驾驶舱”：界面安静、留白充足、视觉层级清晰，有温度但不喧闹。Today Focus 默认占主位，同时 Daily Ritual、Media Journey、Member Hub 作为核心模块出现在首页。

## 2. V1 范围

### V1 必须包含

- PWA 可安装桌面应用。
- 首页生活看板。
- Today Focus 今日焦点。
- Daily Ritual 每日仪式。
- Media Journey 精神食粮。
- Member Hub 会员资产。
- Review 回顾页。
- 文字输入和语音转文字。
- 本地数据保存。
- 数据导出和导入备份。
- 桌面和手机响应式适配。
- 空状态、错误状态和不可用状态的优雅处理。

### V1 暂不包含

- 账号系统。
- 云端同步。
- AI 自动整理任务。
- 系统级桌面小组件。
- 真正的系统通知推送。
- 邮箱、账单、短信或支付记录自动读取。
- 书籍、剧集、订阅的外部数据库自动识别。
- 复杂统计报表、趋势预测和回顾报表导出。
- 看板内完整播放音乐。

### 后续优先路线

- V1.1：接入 Supabase 云同步。
- V1.2：固定时间提醒和浏览器通知。
- V2：AI 自动整理任务、系统级桌面小组件、更多外部内容服务。

## 3. 页面结构

### Dashboard 首页

每天最常打开的页面。

- 顶部：日期、问候语、每日精力状态选择。
- 主区：Today Focus，展示 1 / 3 / 5 个焦点任务。
- 每日仪式：今日一首歌、今日英语学习。
- 辅区：Media Journey 当前卡片、Member Hub 预算与近期续费。
- 今日回顾：今日完成情况和温和总结。

### Tasks 页面

管理全部任务，避免首页变成杂乱清单。

- 添加任务。
- 文字输入和语音转文字。
- 按类型、重要程度和完成状态组织任务。
- 管理灵感收件箱。
- 查看今日已完成任务。

### Ritual 页面

管理每日仪式内容。

- 今日一首歌。
- 网易云音乐跳转播放。
- 备用歌单管理。
- 今日英语学习。
- 英语素材管理。

### Media 页面

管理精神食粮。

- 添加、编辑、删除内容卡片。
- 管理分类、封面、进度、状态和备注。
- 书籍支持作者、页数和灵感记录。

### Members 页面

管理会员订阅资产。

- 添加、编辑、删除订阅。
- 查看本月预算。
- 查看未来 30 天即将续费。
- 停用不再计入预算的订阅。

### Review 页面

做轻量生活回顾。

- 本周回顾。
- 本月回顾。
- 自定义日期区间回顾。

桌面端采用左侧轻导航，手机端采用底部导航。

## 4. Today Focus

### 目标

Today Focus 帮用户从脑内杂事中看见“今天最值得做的几件事”。它不展示所有任务，只展示今日焦点。

### 任务字段

- 内容。
- 类型：生活杂事、长期目标、自我照顾、灵感收件箱。
- 重要程度：低、中、高。
- 创建时间。
- 是否进入今日。
- 完成状态。
- 完成时间。

### 每日精力状态

用户每天选择一个精力状态：

- 低：今日焦点最多 1 件。
- 正常：今日焦点最多 3 件。
- 高：今日焦点最多 5 件。

### 焦点规则

- 高重要程度优先。
- 标记为“进入今日”的任务优先。
- 低精力状态下，自我照顾任务适当提前。
- 灵感收件箱默认不进入今日焦点，除非用户手动转为任务。

### 视觉规则

- 高重要任务更醒目，位置更靠前。
- 中重要任务保持常规层级。
- 低重要任务更安静。
- 已完成任务进入“今日已完成”，视觉淡化但保留成就感。

### 输入体验

- 首页支持快速添加一句任务。
- Tasks 页面支持补充类型和重要程度。
- 语音转文字结果先进入输入框，由用户确认保存。
- 语音不可用时自动回到文字输入。

## 5. Daily Ritual

### 目标

Daily Ritual 是每天的轻量精神入口，不做成音乐播放器或完整学习应用。

### 今日一首歌

- 首页展示歌名、歌手、封面和简短推荐语。
- 点击后跳转到网易云音乐播放页。
- 优先尝试接入网易云音乐推荐或搜索能力。
- 接入失败时使用备用歌单轮换。
- 不在看板内直接播放完整音乐。

### 备用歌单

用户可以手动添加：

- 歌名。
- 歌手。
- 封面链接。
- 网易云链接。
- 备注或推荐语。

### 今日英语学习

- 首页每天展示一条轻量英语内容。
- 内容类型可以是短句、表达、单词组或例句。
- V1 内置一批英语内容。
- 用户可以手动补充自己的英语素材。
- 不做复杂背诵、打卡和复习曲线。

### 推送边界

V1 不做系统通知。每天内容在用户打开看板时展示。固定时间通知放到后续版本。

## 6. Media Journey

### 目标

Media Journey 记录正在吸收、陪伴和消化的内容。它是精神食粮库，不是复杂影音资料库。

### 分类

- 书籍。
- 剧 / 电影。
- 课程。
- 播客。
- 文章。
- 其他。

### 通用字段

所有分类共用：

- 名称。
- 分类。
- 封面链接。
- 进度。
- 状态：进行中、暂停、完成。
- 备注。
- 最近更新时间。

### 书籍额外字段

书籍类型额外支持：

- 作者。
- 总页数。
- 当前页数。
- 阅读进度自动计算。
- 灵感记录。

### 书籍灵感记录

每本书可以记录多条灵感。每条灵感包含：

- 内容。
- 记录日期。
- 可选页码。

### 首页展示

- 展示 1-3 张当前精神食粮卡片。
- 进行中优先。
- 最近更新优先。
- 长期没有推进时显示温和提示。

### 暂不做

- 自动搜索封面。
- 外部内容库接入。
- 复杂评分系统。

## 7. Member Hub

### 目标

Member Hub 帮用户知道每个月有哪些订阅在自动扣钱，哪些快续费了。

### 订阅字段

- 名称。
- 金额。
- 计费周期：每月、每年、自定义。
- 下次续费日期。
- 分类：影音、工具、学习、生活、其他。
- 备注。
- 是否启用。

### 首页展示

- 本月订阅预算总额。
- 最近 1-3 个即将续费项目。
- 临近续费时卡片边框出现轻微呼吸提醒。
- 距离续费越近，提醒越明显，但不能刺眼。

### Members 页面

- 添加、编辑、删除订阅。
- 按续费日期排序。
- 支持停用订阅。
- 显示本月总预算。
- 显示未来 30 天即将续费。

### 预算计算

- 每月订阅计入本月预算。
- 每年订阅只有在本月续费时计入本月预算。
- 自定义订阅根据下次续费日期判断是否计入本月。
- 停用订阅不计入预算。

## 8. Review

### 目标

Review 帮用户看见生活推进的痕迹，不做商业报表式数据面板。

### 首页今日回顾

首页展示：

- 今日完成任务数。
- 今日焦点完成数。
- 今日精力状态。
- 今天是否推进精神食粮。
- 今天是否有订阅续费提醒。
- 一句温和总结。

### Review 页面范围

支持三种回顾范围：

- 本周。
- 本月。
- 自定义日期区间。

### 指标

每个范围展示：

- 完成任务数量。
- 焦点完成率。
- 各任务类型分布。
- 精神食粮推进数量。
- 精神食粮完成数量。
- 书籍灵感记录数量。
- 区间内订阅预算。
- 区间内即将续费项目。

### 图形表达

V1 使用轻量图形：

- 小型进度条。
- 简单环形或条形视觉。
- 数字摘要卡片。
- 短句总结。

不做复杂筛选、趋势预测和回顾报表导出。

## 9. 数据保存与备份

### V1 数据策略

V1 默认使用浏览器本地保存。这样不需要账号，打开即可使用。

保存内容包括：

- 任务。
- 每日精力状态。
- 精神食粮卡片。
- 书籍灵感记录。
- 会员订阅。
- 备用歌单。
- 英语学习素材。

### 空间控制

- 封面优先使用图片链接。
- 本地数据只保存文字、日期、数字和图片 URL。
- 不把大量图片文件直接存进浏览器。

### 备份

V1 支持：

- 导出全部数据为备份文件。
- 从备份文件导入恢复。

### 云同步预留

后续可以接入 Supabase：

- 数据库存储任务、订阅、阅读进度和英语素材。
- Storage 存储用户上传封面。
- Auth 做登录。
- 权限规则保证每个用户只能访问自己的数据。

V1 会把数据读写层单独封装，方便未来替换为 Supabase。

## 10. PWA 桌面路线

V1 做成可安装 PWA：

- 支持从桌面图标启动。
- 有独立窗口体验。
- 支持离线打开基础界面和本地数据。
- 首页做成应用内小组件感看板。

系统级桌面小组件不作为 V1 承诺。后续如果需要真正系统小组件，再评估 Tauri、Electron 或平台原生扩展。

## 11. 错误与边界状态

V1 需要优雅处理：

- 第一次打开，没有任何数据。
- 没有今日焦点。
- 语音输入不可用。
- 网易云接入失败。
- 没有备用歌单。
- 没有英语素材。
- 没有封面。
- 订阅日期过期。
- 金额输入不合法。
- 页数输入不合法。
- 自定义日期区间无数据。
- 手机屏幕较窄。

所有错误状态都应该给用户明确下一步，而不是只显示错误。

## 12. 测试重点

每个阶段都需要测试：

- 刷新页面后数据是否保留。
- 添加、编辑、删除任务是否正确。
- 今日焦点数量是否随精力状态变化。
- 低精力状态下自我照顾任务是否适当提前。
- 语音不可用时是否能手动输入。
- 网易云失败时是否能使用备用歌单。
- 英语内容是否能按日期轮换。
- 书籍页数是否能正确计算进度。
- 灵感记录是否能正确保存。
- 会员预算是否计算正确。
- 续费提醒是否按日期变化。
- 自定义区间回顾是否统计正确。
- 数据导出和导入是否能恢复完整数据。
- 桌面和手机布局是否都可用。
- PWA 是否可安装并能从桌面启动。

## 13. 复杂度评估

整体复杂度：中等偏复杂，但可控。

复杂点主要来自：

- 模块多。
- 视觉完成度要求高。
- 本地数据模型需要提前设计好。
- 统计回顾需要保证口径清楚。
- PWA 和语音能力需要处理浏览器兼容。

风险控制方式：

- 每个模块先做手动管理和精致呈现。
- 不在 V1 引入账号、云同步和 AI 自动整理。
- 外部服务失败时必须有备用方案。
- 先把数据层设计清楚，方便未来升级。
